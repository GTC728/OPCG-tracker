# V4.22 — UX Polish & Conflict Auto-Merge

**Version:** 4.22.0  
**Status:** Shipped  
**Schema:** 20 (unchanged)

## Problem

Seven UX issues surfaced from normal group usage (screenshots, v4.21.0):

1. **False sync conflict on workspace switch** — Switching groups and returning showed「檔案連結變更」for a bookmarked player even though the user never edited links manually.
2. **Record assignment dock** —「分配」section expanded by default, wasting vertical space on mobile.
3. **Stats overview weekly win rate** — Group overview showed a personal weekly chart; win rate belongs on the linked player profile, not the group-wide summary.
4. **Meta transfer chart gaps** — Weeks with zero matches rendered as 0% pillars, creating visual断层 between active weeks.
5. **First/second win rate UI** — Bars did not render (label key mismatch); duplicate cards below both labeled「後攻」.
6. **Operation history** — Mobile users could not see the history panel or undo recent match completions.
7. **Group lobby** — Raw group secret key was displayed; user wants a game-lobby model (public browse, join requests, searchable handle) — larger than one release.

## Research & design principles

| Topic | Research conclusion | Our choice |
|-------|---------------------|------------|
| Benign sync drift | CRDT/LWW systems auto-merge metadata-only deltas when both sides agree on entity identity | `isBenignPlayerJoinDrift` — profile-link-only drift for bookmarked roster identity |
| Weekly win rate scope | Dashboard KPIs = group aggregates; trend lines = individual performance | Remove from Stats overview; keep on `PlayerProfileHub` |
| Sparse time-series | Gapless charts mislead; compress x-axis to active periods or bridge with nulls | Compress meta chart to weeks with `total > 0` |
| Undo in multi-user | Full command-stack undo needs server authority; local undo OK for own recent writes | Expose existing `match_complete` undo on mobile; document V5 event-sourcing for edit/delete |
| Game lobby | Industry pattern: discoverable rooms + join policy + admin console; secrets are implementation detail | Phase 1: hide key, show stats + `@slug`; Phase 2 (V5): visibility, join requests, role model |

**No duplicate systems** — same `detectJoinConflicts`, same `buildFirstSecondStats`, same `OperationHistoryPanel` undo path.

## Architecture

### 1. Benign profile-link auto-merge (workspace switch)

**Root cause:** `switchWorkspace` → `tryAutoRelinkGroupProfile` stamps `linkedUserId` locally before `initializeGroupCollab`. Remote `sync_players.linked_user_id` is often still `null`. `detectJoinConflicts` treated this as a material「檔案連結變更」conflict.

**Fix:**

```
initializeGroupCollab(groupCode)
  → joinCtx = { cloudUserId, bookmarkPlayerId from groupProfileLinks }
  → detectJoinConflicts(snapshot, remote, joinCtx)
      → skip if isBenignPlayerJoinDrift(local, remote, joinCtx)
  → applyJoinMerge uses mergeBenignJoinedPlayer (keeps local linkedUserId)
```

Benign drift rules:

- Only `profileLinkChanged` (aliases ignored)
- Bookmarked player matches `bookmarkPlayerId`
- One side `linkedUserId` is null or matches `cloudUserId`

Users no longer see a manual conflict panel for normal group round-trips.

### 2. Stats overview scope

- **Removed** `WeeklyWinRateChart` from Stats overview tab.
- **Kept** weekly win rate on linked player profile (`PlayerProfileHub`).

Overview now shows group-level KPIs only (total matches, avg duration, meta diversity, etc.).

### 3. Meta transfer chart — active weeks only

`buildMetaTransferChartPoints` filters to `week.total > 0` before building stacked areas. X-axis spans only weeks with recorded matches — no zero-height断层.

Hint copy: `stats.metaTransferHint` (4 locales).

### 4. First/second win rate — unified card

- `buildFirstSecondStats` labels: `'first'` / `'second'` (was `'先攻'` / `'後攻'`).
- `FirstSecondSection`: single card with two labeled progress bars + inline W/sample text.
- Removed duplicate stat grid that mislabeled both cards as「後攻」.

### 5. Record page — assignment dock collapsed

- `TableBoard` / `AssignmentDock`: `drawerExpanded` default `false`.

### 6. Operation history on mobile

- `OperationHistoryPanel` visible on all breakpoints (removed `hidden md:block`).
- `SystemStatusPanel` deduplicated — no parallel mobile-only audit list.
- Existing undo: `match_complete` entries still offer revert; edit/delete undo deferred to V5 (multi-user authority).

### 7. Group lobby — Phase 1 (V5 roadmap for full lobby)

**Shipped in V4.22:**

- Removed plaintext group secret key from admin UI.
- Lobby stats: active players, completed matches, sessions.
- Public handle preview: `@inviteSlug` when set.
- Copy refresh: `groupLobby.descV422`.

**V5.0+ game lobby (not in this release):**

| Capability | Notes |
|------------|-------|
| Discoverability | Search by display name + public group id / `@slug` |
| Join policy | Open / request / invite-only |
| Visibility | Public stats vs members-only |
| Roles | Owner (團長), admin, player — drop standalone「觀察者」(public = read-only browse) |
| Join requests | `group_join_requests` + admin approve/deny |
| SQL | `visibility`, `join_policy`, RPCs — see `docs/V4.16-ROADMAP.md` V5 |
