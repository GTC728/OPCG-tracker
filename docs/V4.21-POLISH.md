# V4.21 — Polish, i18n & Achievement Backlog

**Version:** 4.21.0  
**Status:** Shipped  
**Schema:** 20

## Problem

After V4.16–V4.20, three polish gaps remained:

1. **Achievement backlog** — Several 🟢 metrics in `ACHIEVEMENTS-BACKLOG.md` were stubbed or mis-bound; V4.19–V4.20 features (conflicts, multi-group) had no achievements.
2. **i18n** — Stats page, sync queue, conflict/history diff lines, and win-rate helpers still hardcoded Chinese or English strings.
3. **Roadmap drift** — V4.16–V4.20 items needed consistent “shipped” status in docs.

## Research & design principles

| Topic | Research conclusion | Our choice |
|-------|---------------------|------------|
| i18n in data layer | Keep **stable codes** in lib; translate in UI via `t()` | `entityDiff.*` codes + `formatEntityDiffCodes()` |
| Win-rate copy | Small-sample labels belong in locale files, not inline | `stats.sample.*`, `stats.tooltip.*`, `defaultTranslate` in `winRateDisplay` |
| Achievement metrics | No second event store; extend existing backlog pipeline | Fix `buildBacklogStats` + `evaluateRemainingBacklogMetrics` |
| Circular imports | `achievements.ts` ↔ backlog eval caused init failures | Split `achievementsRemainingDefinitions.ts`; progress stats in remaining eval only |

**No duplicate systems** — same `reconcileAchievementUnlocks`, same `SyncConflict` type, same four locale dictionaries.

## Architecture

### Stable entity diff codes

```
conflictResolver / operationHistory
  → summarize*Diff() returns EntityDiffCode[]
  → UI: formatEntityDiffCodes(codes, t)
```

Codes: `winnerChanged`, `notesChanged`, `fieldsUpdated`, etc. Legacy `diffLines` migrated to `diffCodes` in schema v20.

### Achievement backlog batch (V4.21)

| ID | Metric source | Notes |
|----|---------------|-------|
| `conflict_survivor` | `settings.conflictsResolvedCount` | Incremented in `resolveSyncConflict` |
| `multi_group_tourist` | `settings.groupVisitCodes.length` | Updated on workspace switch |
| `group_anchor` | `groupParticipation` (linked profile + group + matches) | Fixed mis-binding |
| `one_session_five_unlock` | `sameSessionUnlocks` (8h burst window) | Fixed stub |
| `secret_all_dash` | Notes matching `^[-—]{3,}$` | Secret win |
| `secret_lucky_roll` | Notes matching `/roll|骰/i` | Secret win |
| Progress family | `weightedProgressPct`, category max counts | From unlock tiers (remaining catalog) |

### i18n coverage (4 locales)

- `entityDiff.*`, `stats.section.*`, `stats.matchup.*`, `stats.insight.*`
- `stats.sample.*`, `stats.reliability.*`, `stats.tooltip.*`
- `groupRole.*`, `syncQueue.*`
- **StatsPage** — full pass (overview, heatmaps, profiles, insights)
- **SyncQueuePanel**, **GroupConflictPanel**, **OperationHistoryPanel**, **SyncStatusBanner**
- **ShareExportSheet** — first/second labels aligned with `buildFirstSecondStats` (`first` / `second` keys)

### Settings (schema v20)

```typescript
conflictsResolvedCount: number
groupVisitCodes: string[]
```
