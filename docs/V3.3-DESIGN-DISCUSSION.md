**Version:** V3.3 — Match Flow & Density Patch  
**Date:** 2026-07-03  
**Status:** Implemented

---

## English Summary

V3.3 focuses on match-flow polish and information density without changing core data models. Key deliverables: session roster selection, compact Record page layout, combobox-style deck picker, post-match notes sheet, color-coded history winners, player alias chips, stats micro-leaderboards with usage-sorted pie charts, expanded insights, leader locale aliases (zh/ja/yue), and README live demo links. Table/slot-based match management is deferred to V3.4.

## 1. Session 參與者選擇

### 現況
- `SessionPlayer` 與 `getSessionRoster()` 已存在，store 從未寫入 `sessionPlayers`。
- 新對局顯示全部未封存玩家。

### 決策
新建或切換到無 roster 的 Session 時，彈出「本輪參與玩家」多選。預設勾選近 7 天最常出場玩家。roster < 2 時禁用新對局。

### Acceptance Criteria
- [ ] `setSessionRoster(sessionId, playerIds[])` 持久化到 `sessionPlayers`
- [ ] 新對局表單只顯示 roster 内玩家（fallback：從 session 對局推導）
- [ ] Settings → Session 可編輯 roster
- [ ] 玩家 chip 按 `(sessionMatches + recent30d)` 降序

---

## 2. 拖曳 / 固定桌子

### 決策
V3.3 不做完整拖曳。進行中對局改緊湊單行 + 展開。V3.4 規劃 slot-based 桌子模式。

---

## 3. 記錄頁排版

### 決策
- Session 摘要：單行 inline meta，可展開 2×2
- 進行中：collapsed 預設，展開後先攻 + WIN
- 最近對局：session 範圍 3–5 場單行，勝方色條

### Acceptance Criteria
- [ ] Session 摘要預設 ≤ 2 行
- [ ] Active match collapsed 一行摘要
- [ ] 最近完成對局在 Record 頁顯示（session scoped）

---

## 4. 歷史勝方顏色

### 決策
勝方 `border-l-4 border-success` + 名字高亮。共用 `MatchResultRow`。

---

## 5. 玩家別名 Chips

### 決策
Tag chips + × 刪除 + 「+ 新增別名」。Deck 別名同模式。

---

## 6. 完成後備註

### 決策
移除開局備註。WIN → BottomSheet 摘要 + 可選備註 + 略過/儲存。

---

## 7. Deck Combobox

### 決策
選中後 input 顯示 DeckLabel，關閉結果面板，× 清除。推薦僅未選中時顯示 2–3 個不同 deck。

---

## 8. 進行中對局

### 決策
不開放 mid-game 改 player/deck。Deck 兩行 stack，WIN 區高度減半。

---

## 9. 統計

### 決策
- Overview：玩家 Top 5 + 牌組 Top 5（按出場）
- 餅圖：按出場比例降序
- Insights：擴充至 6 類規則，cap 4 條

---

## 10. 歷史編輯

### 決策
垂直 stack：Player A → Deck A → Player B → Deck B → Winner → First → Notes。

---

## 11. GitHub Live Link

### 決策
