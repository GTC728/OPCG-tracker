# V3.4.1 — Record / Table Mode UX Patch

> **狀態**：完成（V3.4.1）  
> **前置**：V3.4.0 Table Mode  
> **目標**：手機一屏至少看見 **2 張桌**，整合分配流程，去掉重複區塊。

---

## 問題摘要（來自 V3.4.0 實際使用）

| 問題 | 現況 |
|------|------|
| 桌子太高 | _drop 區 + 先攻列 + 大 WIN 卡 + 編輯_，一桌約占半屏 |
| 新對局 vs 拖曳分配 | 「＋ 新對局」開 BottomSheet 填表，與拖曳分配功能重疊 |
| 最近完成 vs 最近組合 | 同一對局資訊顯示兩次，只是有無勝方色/時間 |
| Rematch 流程 | 點最近組合只預填表單，不會自動放到空桌 |

---

## 設計原則

1. **Table-first**：Record 頁以「桌子」為中心；建立對局 = 往空桌放人，不是另開一條流程。
2. **Mobile density**：以 iPhone SE～標準 Android（~667–844px 高）為基準，扣除 bottom nav + 精簡分配欄後，**2 桌完整可操作**。
3. **Drag + Tap**：手機上拖曳不穩，**點選分配**與拖曳並存。
4. **Show once**：同一資訊不在 drop 區與 WIN 區重複展示。

## A. 緊湊桌卡（Compact Table Slot）

### 現況高度來源

```
桌標題列
+ 左 drop (min-h-20) | vs | 右 drop (min-h-20)   ← 與下方 WIN 重複顯示名稱/牌組
+ 擲骰 | A先 | B先
+ 兩張大 WIN 卡 (p-3, text-2xl)
+ 編輯玩家/牌組
```

### 目標布局（完整對局時 ~120–140px/桌）

**未完成**（缺玩家或牌組）：

```
┌ 桌 2 ─────────────────────── 清空 ┐
│ [左: 空/半填]  vs  [右: 空/半填]   │  ← 單行 compact drop，min-h ~44px
└───────────────────────────────────┘
```

**已完成**（四人四牌組齊）：

```
┌ 桌 2 ───────── 先:Sim ─ 清空 ┐
│ Simon OP16艾斯  │  Simon OP16艾斯 │  ← 左側 compact，右側 compact
│     [ WIN ]     │     [ WIN ]     │  ← 按鈕高度 ~48px，字 text-lg
│ 擲骰 · A先 · B先 · 編輯 ▼        │  ← 先攻合并为一行 icon/chip
└────────────────────────────────┘
```

### 具體改動

| 項目 | 做法 |
|------|------|
| 去掉 drop + WIN 雙重展示 | 完成後 **隱藏 drop 虛線框**，只留一行對局摘要 + WIN |
| WIN 卡縮小 | `p-2`、`text-lg`、去掉重複 DeckLabel 第二行（名稱旁已有 code） |
| 先攻列 | 改為 chip 列或 ▼ 折疊「進階」 |
| 編輯 | 收到 ▼ 或長按桌卡，不常駐大按鈕 |
| 空桌 | 單行虛線「點此或拖入分配」 |

### 驗收

- [ ] iPhone 12 直向（844×390）：分配欄折疊狀態下，**≥2 張已填滿的桌**同屏可見且 WIN 可點。
- [ ] 文字仍可读：玩家名 truncate、DeckLabel showCode。

## B. 整合分配欄（取代「新對局」）

### 決策：**刪除「＋ 新對局」主按鈕**

建立對局統一為：**分配欄 → 空桌**。

`MatchForm` BottomSheet **保留但降級**：
- 僅在「編輯進行中對局」「Rematch 確認」時使用
- 不再作為 Record 頁主入口

### 新組件：`AssignmentDock`（分配坞）

```
┌ 分配 ─────────────────── [▼ 收合] ┐
│ 玩家                               │
│ [Simon] [Bobby] [Ken] …  → 橫向捲動 │
├────────────────────────────────────┤
│ 牌組                               │
│ [🔍 搜尋 OP/ST/EB/主將…]           │  ← 複用 DeckSearchField（compact）
│ 最近： [OP16艾斯] [EB01漢尼拔] …    │  ← session 級 recent + 可拖/可點
└────────────────────────────────────┘
```

| 區塊 | 行為 |
|------|------|
| **玩家** | 本場 roster chips；**拖**到桌左/右，或 **點玩家 → 點桌側**（tap-assign） |
| **牌組** | 搜尋結果 + 最近牌組 chips；拖到已有玩家的那一側，或 tap-assign |
| 拖玩家 | 自動帶該玩家最近牌組（現有邏輯） |
| 拖牌組 | 只填 deck 側（現有邏輯） |

### Tap-assign 流程（手機必備）

1. 點分配欄中的 **玩家** → 進入「選桌模式」（高亮空桌/半填桌側）
2. 點目標桌的 **左或右** → `assignTableSide`
3. 取消：再點空白或 Esc

牌組同理：先選 deck chip，再點桌側。

### 分配欄可收合

- 預設展開；分配完成後 **自動收合**（可設定）
- 收合後只顯示一行：`分配 · Simon Bobby · 3 牌組` + 展開按鈕
- 省垂直空間給桌子

### 驗收

- [ ] Record 頁無「＋ 新對局」按鈕
- [ ] 牌組區有搜尋 + 最近牌組
- [ ] 玩家/牌組分區清晰
- [ ] 手機可不拖曳，純點選完成分配

## C. 最近組合（合併「最近完成」）

### 決策：**只保留「最近組合」一區**

移除「最近完成」獨立區塊。

### 最近組合列 UI

```
最近對局
┌─────────────────────────────────────────────┐
│ Simon OP16艾斯  vs  Bobby EB01漢尼拔        │
│ 上次：Simon 勝 · 01:03        [ 再戰 ▶ ]    │  ← 可選顯示上次結果
└─────────────────────────────────────────────┘
```

- 資料來源：`getRecentCombos` + 可選 join 最近一場同 combo 的 match（勝方、時間）
- **不再**用假 match 畫 MatchResultRow（去掉误导性勝方色）

### Rematch 流程

1. 點列或「再戰」→ 開 **RematchSheet**（精簡 MatchForm）
   - 預填四人四牌
   - 可改牌組、交換左右、設先攻
   - **無**「確認建立」後扔進 overflow——改為 **「放到空桌」**
2. 確認 → `findFirstEmptyTableSlot` → `createActiveMatch({ ...input, tableSlot })`
3. 若無空桌 → toast「請先清空一桌或增加桌數」+ 可選自動 +1 桌

### 驗收

- [ ] 無「最近完成」區
- [ ] 最近組合可顯示 optional 上次結果
- [ ] 確認後對局出現在**第一個空桌**，不在 unassigned
