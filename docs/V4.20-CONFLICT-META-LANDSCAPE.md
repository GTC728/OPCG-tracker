# V4.20 — Conflict, Meta & Landscape

**Version:** 4.20.0  
**Status:** Shipped  
**Roadmap IDs:** G-03, S-02, R-04

## Problem

Three gaps remained after V4.19:

1. **G-03** — Group join/pull used silent LWW; concurrent edits could lose local changes without user awareness.
2. **S-02** — Meta summary showed static deck share only; no time dimension for meta shifts.
3. **R-04** — Record page on iPad landscape stacked tables above assignment dock instead of side-by-side.

## Research & design principles

| Topic | Research conclusion | Our choice |
|-------|---------------------|------------|
| Conflict resolution | CRDTs for collaborative text; LWW acceptable for low-conflict entity sync when conflicts are rare | **LWW default + explicit UI** for true conflicts only |
| Conflict UX | Expose surprises; offer keep mine / keep theirs; avoid dialog storms | Queue conflicts; banner + settings panel |
| Meta charts | 100% stacked area for composition; 5–7 categories; stable bottom layer; y-axis from zero | Top 5 decks + Other; `stackOffset="expand"` |
| Tablet layout | Side-by-side when width + landscape; widen shell | `768px + landscape` query; `max-w-5xl` shell |

**No new event store** — conflicts reuse existing entity payloads and `local*Touch` maps in `groupSync.ts`.

## Architecture

### G-03 — Conflict merge UI

```
pull / join
    ↓
detect: localTouch > 0 AND remoteAt >= localTouch AND payload differs
    ↓
skip silent apply → queue SyncConflict (local + remote snapshots)
    ↓
GroupConflictPanel / SyncStatusBanner
    ↓
resolveSyncConflict('local' | 'remote')
    ↓
reinforceLocalEntityTouch OR applyRemote + clear touch
```

**Join merge** (`applyJoinMerge`):

- Non-conflicting remote entities apply normally.
- Conflicting ids keep **local** snapshot until user resolves.
- Local-only entities (not on server) are preserved.

**Pull merge**:

- Non-conflicting rows apply per LWW.
- Conflicting rows keep local; queued for review.

Files:

| File | Role |
|------|------|
| `src/lib/conflictResolver.ts` | Detection, diff, resolution apply |
| `src/lib/groupSync.ts` | `fetchRemoteGroupEntities`, join/pull integration |
| `src/components/settings/GroupConflictPanel.tsx` | Resolution UI |
| `src/stores/appStore.ts` | `resolveSyncConflict`, clear on workspace switch |

### S-02 — Meta transfer chart

- `buildWeeklyDeckMetaStats()` — deck appearances per week (both sides of each match).
- `buildMetaTransferChartPoints()` — stable deck keys, count-based stacks.
- `MetaTransferChart` — 100% stacked `AreaChart` on Stats overview.

Reuses: `getWeekStart`, `formatWeekLabel`, `getChartSliceFill`, scoped matches from `useScopedStats`.

### R-04 — iPad landscape

- `useMediaQuery('(min-width: 768px) and (orientation: landscape)')` in `TableBoard`.
- Grid: tables left (2-column table grid), `AssignmentDock` sticky right (`ASSIGNMENT_SIDE_PANEL_WIDTH`).
- `AppShell` widens to `max-w-5xl` in landscape tablet.

## Non-goals

- Field-level three-way merge (deferred — entity-level sufficient for match tracker).
- CRDT / PowerSync introduction.
- Server-side conflict persistence.
