# V5.0 — Game Lobby & Operation Undo

**Version:** 5.0.0  
**Status:** Shipped  
**Schema:** 20 (unchanged)  
**SQL:** `docs/supabase-v5.0-group-lobby.sql` (required for full lobby features)

## Summary

V5.0 delivers the complete game-lobby model discussed after V4.22: public discovery, join policies, admin console, role model refresh, invite links, and extended local operation undo.

## Research & design

| Pattern | Industry reference | OPCG choice |
|---------|-------------------|-------------|
| Discoverability | PlayFab/Unity lobby `AccessPolicy` + query filters | `search_public_groups` RPC; compress to active public groups only |
| Join policy | Open / request / invite-only | `join_group_with_policy` SECURITY DEFINER — no raw `group_members` insert |
| Admin hierarchy | owner > admin > member (3 levels) | `admin` role; `reader` migrated to `member`; public browse replaces viewer role |
| Multi-user undo | Only revert own writes | `isLocalAuditActor` gates edit/delete/complete undo |
| Stats at scale | Denormalized lobby snapshot | `stats_snapshot` + `refresh_group_stats_snapshot` from `sync_*` counts |

**No duplicate systems** — extends `groupRegistry`, `groupPermissions`, `WorkspaceHub`, `GroupLobbyPanel`; new `groupLobby.ts` client facade over V5 RPCs.

## Architecture

### Server (Supabase)

**`groups` extensions:** `public_id`, `visibility`, `join_policy`, `description`, `stats_snapshot`, `last_active_at`

**New tables:**
- `group_join_requests` — pending/approved/rejected
- `group_invite_links` — token, expiry, max uses

**Key RPCs:**
- `search_public_groups` — browse with sort (active / matches / players)
- `resolve_group_lookup` — by public_id, @slug, or name
- `join_group_with_policy` — open join / request / invite token
- `list_group_join_requests` + `review_join_request`
- `transfer_group_ownership`
- `update_group_lobby_settings`
- `create_group_invite_link`
- `refresh_group_stats_snapshot`

**RLS:** Public read on `visibility = 'public'` groups; managers (owner/admin) update profiles and members; self-join policy removed — joins go through RPC.

### Client

| Module | Role |
|--------|------|
| `groupLobby.ts` | Search, join, requests, settings, invite links |
| `GroupLobbyHub.tsx` | Browse list + detail + join CTA |
| `GroupLobbyPanel.tsx` | Admin: policy, requests, transfer, invite |
| `GroupMembershipPanel.tsx` | Search-by-ID join (no 8-char secret UX) |
| `WorkspaceHub.tsx` |「探索群組大廳」entry |
| `operationHistory.ts` | Undo kinds: complete / edit / delete (local only) |

### Roles (V5)

| Role | Record | Delete matches | Manage lobby | Transfer owner |
|------|--------|----------------|--------------|----------------|
| owner | ✓ | ✓ | ✓ | ✓ |
| admin | ✓ | ✓ | ✓ | ✗ |
| member | ✓ | ✗ | ✗ | ✗ |

## Deployment

1. Run `docs/supabase-v5.0-group-lobby.sql` on Supabase (after v4.18 SQL).
2. Deploy client 5.0.0.
3. Existing groups: `public_id` backfilled from `storage_code`; `reader` → `member`.

Without SQL migration, client falls back to V4 join (`joinGroupLegacy`) and V4 registry update for lobby settings.
